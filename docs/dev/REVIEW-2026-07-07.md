## P1 正确性修复（先做）

1. ★〔B|Codex〕**custom_data_path 所有权泄漏**（高）：`commands.rs::update_settings` 无差别 `settings.set(settings)`，设置页保存/重置可改写 `custom_data_path` 而不迁移数据库（重置发 `customDataPath: null`，重启后静默换目录丢下旧库）。修：update_settings 里强制保留 `old_settings.custom_data_path`（该字段只允许 `migrate_data_location` 修改），前端重置 payload 同步带上当前值。
2. ★〔B|Codex〕**capture_paused 所有权泄漏**（高）：设置窗口陈旧对象保存会覆盖托盘"暂停采集"；且 update_settings 改变该值时不重建托盘（勾选态失真）。修：update_settings 强制保留 `old_settings.capture_paused`（只允许托盘命令改）；或纳入 tray-rebuild 条件。
3. 〔B|R2〕**单实例回调非主线程调 AppKit**（高，插件 2.4.2 源码已核实回调跑在 tokio worker）：`main.rs` single_instance 回调直调 `window::show_quickbar`（NSWorkspace/orderFront/invalidateShadow 全是 main-thread-only）。修：回调体包 `app.run_on_main_thread(move || …)`。
4. 〔A〕**MarkdownContent 整套样式失效**：`ui/MarkdownContent.svelte` 大量 `hsl(var(--border))`，而 app.css token 已是完整 `hsl(...)` → `hsl(hsl(...))` 非法整条丢弃（h1/h2 边框、code/pre/blockquote/th 背景、链接色全失效）。修：全部改直接 `var(--x)`。
5. 〔A〕**粘贴/复制失败完全静默**：`clipboard.svelte.ts` useClip/useSelectedClips 的 catch 里 `mode === 'copy'` 永假（调用方只产 default/opposite）。修：失败一律 toast（文案按 autoPaste 区分）；同时把前端 `PasteMode` 收窄为 `'default'|'opposite'`（Rust 侧 mode 枚举保留）。
6. 〔B|R2〕**env_logger 强制 Debug**：`main.rs` `filter_level(Debug)` 压制 RUST_LOG 且 release 恒 debug。修：`Builder::from_env(Env::default().default_filter_or(cfg!(debug_assertions).then…))` 模式，仅未设 RUST_LOG 时给默认（debug 构建 debug、release info）。
7. 〔B|R1〕**prune 两条 DELETE 依赖无保证的并列排序**：`storage.rs::prune_history_with_conn` 两次 `ORDER BY timestamp DESC` 无 id 决胜键，并列时行集可分叉产生 FTS 孤儿（fts_needs_rebuild 会自愈故低危）。修：两处补 `, id DESC`。
8. ★〔B|Codex〕**duplicate 刷新不更新 source_app**：`refresh_duplicate_with_conn` 只更 timestamp+html，UI 显示旧来源。修：加 `source_app = COALESCE(?, source_app)` 并传 `item.source_app`。
9. 〔B|R1〕**MAX(pin_order) 吞真实 DB 错误**：`storage.rs::update_pin_with_conn` `.unwrap_or(None)`。修：改 `?`。
10. 〔C|R3 置信85%〕**面板切换 anchor effect 用旧锚覆盖重置**：`+page.svelte` resetPanelAndReveal 的 `flushSync()` 让 anchor effect 带旧 `anchoredId` 把 selectedIndex 恢复到旧行，而视口滚到第 0 行 → 高亮在屏外、Enter 粘不可见行。修（先写复现测试再动）：reset 后 flushSync 前 `anchoredId = null`。
11. 〔B|R3〕**system 主题不监听 prefers-color-scheme 变化**：syncTheme 只读一次。修：theme==='system' 时 addEventListener('change') + cleanup。
12. 〔B|R3 已编译验证〕**`!hover:bg-green-700` 不生成 CSS**：AboutSection.svelte 安装按钮。修：`hover:!bg-green-700`；同文件 `updateMessage.includes('✓')` 死条件一并清。
13. 〔B|Codex〕**迁移探针可覆盖用户文件**：migration.rs 固定名 `.clipman_test` 直接 write。修：UUID 临时名 + `create_new(true)`。
14. 〔B|R1〕**paste_clips 全图片路径不收 QuickBar**：merged_count==0 早退跳过 hide_quickbar，面板留在原地无反馈。修：hide 提前或返回可提示错误。
15. 〔B|Codex〕**max_image_dimension 溢出回绕**：settings.rs `v as u32` 在 clamp 前。修：先按 u64 clamp 再转。
16. 〔B|R3〕**isLoading 可卡死**：loadHistory 的 finally 只在非搜索态清 isLoading（启动瞬间输入搜索词触发）。修：search 非空路径成功后补清。

## 两模型一致的正面评价（保持勿破坏）

CopyMarker/D5 不变量纪律与测试钉死、键集分页三件套（双列游标+索引+EXPLAIN 回归测试）、FTS 卫生（双条件 join+自检自愈）、启动降级链纯函数化+失败注入测试、前端并发防御体系（RequestSequencer/修订号重放/竞态测试）、"为什么"注释文化（Tahoe TCC、incremental_vacuum 逐行 drain 等一手经验）。
