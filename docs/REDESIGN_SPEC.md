# ClipMan 体验重构规格（v2.0.0）

> 目标读者：项目作者。本文讲清楚「要做成什么样」和「为什么」。技术细节集中在带 🔧 的小节，看不懂可先跳过。
> 配套文档：实现步骤见 [`DEVELOPMENT_PLAN.md`](./DEVELOPMENT_PLAN.md)。

---

## 0. 一句话目标

把 ClipMan 从「一个能查历史的窗口」变成「按一下快捷键、选一条、自动粘进当前应用」的取用工具——**让一次粘贴从 5 步降到 1～2 步**；同时把「常用置顶」做成稳定、可命名、能形成肌肉记忆的一等公民；并砍掉收益≈0 的加密层，让代码与搜索都变干净。

---

## 1. 背景：现在卡在哪

用户用剪贴板管理器只有一个根本动作——**把历史内容粘贴进当前应用**。但现在这条链路是断的：

| 现状步骤 | 问题 | 代码位置 |
|---|---|---|
| 1. 按 `Cmd/Ctrl+Shift+V` | 弹出 800×600 带标题栏大窗，出现在上次位置 | `tauri.conf.json:13-25`，`main.rs:281-285` |
| 2. 鼠标移到条目 | 无任何键盘导航 | `+page.svelte`（无 keydown） |
| 3. 悬停→点小复制按钮 | 整张卡片不可点，按钮要悬停才出现 | `ClipboardItem.svelte:168-183` |
| 4. 手动切回原应用 | 窗口不自动隐藏，挡着 | `main.rs:252-258` |
| 5. 手动按 `Cmd/Ctrl+V` | 真正目的，要用户自己做 | — |

**5 步、两次「键盘↔鼠标」切换。** 目标压到：`快捷键 →（可选打几个字）→ 回车，自动粘好`。

---

## 2. 范围

### 2.1 平台
- **一流支持：macOS、Windows**（自动粘贴；macOS 非激活面板不抢焦点，Windows 记录并恢复原前台窗口）。
- **尽力而为：Linux**（X11 用 `enigo` 模拟；Wayland 受限时自动降级为「仅复制」，不报错）。

### 2.2 本期做
1. 轻量弹窗 QuickBar（无边框、macOS 不抢焦点 / Windows 快速恢复焦点、唤起即用、选完即消失）。
2. 全键盘驱动 + 整卡片可点 + 选中高亮。
3. 自动粘回原应用（按入口区分，第 5 节）。
4. **置顶/常用一等公民**：双面板、稳定数字槽位、可命名标签（第 7 节）。
5. 去掉 AES 加密层 + 数据层简化 + 一次性迁移。
6. 真·FTS5 搜索（trigram，中文可用）。
7. 设置/管理拆为独立普通窗口。
8. 跳过密码类剪贴板（替代加密的隐私正解，最小集）。

### 2.3 本期不做（明确划界，避免范围蔓延）
- 多设备同步、插件系统、命令行工具。
- 分组/文件夹：**仅在数据库预留字段**，不实现 UI。
- 单条常用项绑专属全局快捷键（TextExpander 式）：数据预留，不实现。
- 真·落盘加密（将来若做，正确姿势见 §10.4）。
- 富文本/HTML 保真粘贴（继续按纯文本）。

## 3. 设计原则

1. **键盘优先**：所有高频操作不碰鼠标即可完成。
2. **不丢目标应用**：macOS 用非激活面板让目标应用始终前台；Windows 允许弹窗短暂取焦点，但必须记录原前台窗口并在粘贴前恢复。
3. **选完即走**：取用后窗口立刻消失。
4. **按入口分流**：同一动作在不同入口有不同默认行为，匹配不同习惯（第 5 节）。
5. **常用是一等公民**：不是历史的附属 tab，而是有稳定槽位、可命名的精选库。
6. **少即是多**：能用系统能力解决的（钥匙串、剪贴板类型标记、SQLite 索引），不自造复杂机制。
7. **后端单一数据源**：前端只负责展示，数据规则全在后端，避免前后端各写一遍。

## 4. 核心一：三个入口如何分工

「自动粘贴」要成立，前提是**知道粘到哪个应用**。三个入口此前提强弱不同，故默认行为不同：

| 入口 | 触发 | 用户心态 | 目标应用明确？ | 选中后**默认**行为 |
|---|---|---|---|---|
| **① QuickBar 弹窗** | 键盘 `Cmd/Ctrl+Shift+V` | 「我正在输入框里，现在就要粘」 | ✅ 当前前台应用 | **自动粘贴**（可关） |
| **② 托盘菜单** | 鼠标点菜单栏图标 | 顺手浏览，偶尔取用 | ❌ 焦点已模糊 | **仅复制**（保持现状） |
| **③ 设置/管理窗口** | 托盘→设置 / 弹窗齿轮 | 管理数据、改配置 | — 不涉及粘贴 | 点条目=复制 |

- **统一开关** `auto_paste`（默认开）：开=入口①自动粘；关=入口①退化为「复制+关窗」。入口②③不受影响，永远仅复制。
- **临时反转**：弹窗内 `Enter`=默认行为；`Cmd/Ctrl+Enter`=相反行为（默认自动粘时它=仅复制/粘为纯文本）。

## 5. 核心二：QuickBar 弹窗交互规格

复用现有「main」窗口彻底改造形态（macOS 为非激活面板；Windows 为可取焦点的轻量弹窗，取用时恢复原前台窗口）。

### 5.1 外观与定位
- 无边框（`decorations:false`）、不可改大小、置顶（`alwaysOnTop:true`）。
- 尺寸 **560×420**（v1 固定，后续微调）。
- 出现在**当前活动屏幕**，水平居中、垂直略偏上（类 Spotlight）。光标精确定位作后续微调。
- 启动默认隐藏，仅快捷键唤起。

### 5.2 唤起与关闭
- 唤起即**自动聚焦搜索框**，可立即打字。
- 关闭（任一）：`Esc`（v1 直接关窗）/ **失焦自动隐藏** / 完成取用后自动隐藏。

### 5.3 双面板：最近 / 常用
- 打开默认在**「最近」**（最高频场景，最近项就在最顶）。
- `Tab` 一键切到**「常用」**面板；常用项带固定 `1～9` 序号，稳定不变。
- 打字搜索**同时命中最近 + 常用**（含标签），不分面板。
- （可选）第二个全局快捷键 `pinned_shortcut` 直达「常用」面板（默认不绑，设置里可设）。

### 5.4 键盘操作

| 按键 | 行为 |
|---|---|
| 直接打字 | 即时过滤搜索（无需先点搜索框） |
| `↑` / `↓` | 移动选中项，列表自动滚动跟随 |
| `Tab` | 切换 最近 ↔ 常用 面板 |
| `Enter` | 对选中项执行**默认行为**（弹窗=自动粘贴） |
| `Cmd/Ctrl+Enter` | 执行**相反行为**（仅复制 / 粘为纯文本） |
| `1`～`9` | 直接取用当前面板第 N 条 |
| `Cmd/Ctrl+P` | 置顶/取消置顶 当前项（在最近→提拔为常用） |
| `Cmd/Ctrl+Shift+↑/↓` | （常用面板）调整选中项顺序 |
| `Cmd/Ctrl+Delete` | 删除当前项 |
| `Esc` | 关闭 |

### 5.5 鼠标操作（同时保留）
- **整张卡片可点 = 取用**（修掉「只有悬停小按钮可点」的反人类设计）。
- 卡片右侧悬停显示「置顶/删除/编辑标签」小按钮。
- 鼠标悬停项也成为「当前选中项」，与键盘选中统一。

### 5.6 选中态视觉
- 当前选中项明确高亮（背景 + 左侧强调边）。
- 列表项左侧显示 `1～9` 序号（前 9 条）。

## 6. 核心三：自动粘贴技术方案 🔧

核心难点：**取用时必须把内容粘回原应用**。macOS 通过非激活面板让焦点全程留在原应用；Windows 通过「唤起前记录原前台窗口、取用时恢复」实现同等用户结果。

### 6.1 macOS（非激活面板）
- 把弹窗变成 **non-activating NSPanel**（`NSWindowStyleMaskNonactivatingPanel` + floating level + 合适 `collectionBehavior`）：能收键盘输入，但**不让 ClipMan 变前台**，原应用始终保持前台。Raycast/Alfred 同款。
- 借助社区插件 **`tauri-nspanel`** 把 Tauri 窗口转成 NSPanel，免手写大量 ObjC。
- 取用：①写剪贴板 → ②`enigo`/CGEvent 发 `Cmd+V` → ③隐藏面板。原应用一直前台，粘贴直接落它。
- 权限：模拟按键需「辅助功能」，而读剪贴板本来就需要，无新增授权负担。未授权时检测并明确引导。

### 6.2 Windows（记录并恢复前台窗口）
> ⚠️ 不采用 `WS_EX_NOACTIVATE`：MS 文档明确该样式的顶层窗口**不会成为前台窗口**，这与「搜索框要能直接接收键盘输入」天然冲突。所以 Windows 改用「记录→恢复」模型（落地前先做 spike，见计划 WP-0.0）：
- 弹窗是**正常可取焦点**窗口（`WS_EX_TOOLWINDOW` 不进任务栏即可）。
- 唤起前 `GetForegroundWindow()` 记下原前台窗口句柄；弹窗正常拿焦点供输入。
- 取用：①写剪贴板 → ②隐藏弹窗 → ③`SetForegroundWindow(原窗口)`（受系统限制，必要时 `AttachThreadInput` 绕前台锁）→ ④`enigo` 发 `Ctrl+V`。
- **与 macOS 的关键差异**：macOS 焦点全程没离开原应用；Windows 是「短暂取焦点→恢复」。WP-1.B 的粘贴流程要按平台分别处理。

### 6.3 跨平台公共
- 按键模拟统一用 **`enigo`**，平台相关只剩「窗口不抢焦点」一小块。
- **防自捕获**：把 `last_copied_by_us` 从 `Option<String>`（仅文本）升级为**基于规范化剪贴板 payload 哈希的标记** `Option<CopyMarker>`（payload SHA256 + 类型）。文本 payload = UTF-8 bytes；图片 payload = 剪贴板读写两侧共用的规范化表示（如 `width + height + RGBA bytes`），**不要用 DB 里 PNG 原图字节直接做 marker**，否则写入剪贴板后再读回的图片重编码会导致哈希不一致。写剪贴板时设置，`clipboard.rs` 收到变化时按同一规则对来料算哈希比对、命中则跳过——**文本与图片统一覆盖**。该类型是跨文件契约，在 Phase 0 统一定义（见计划 WP-0.1/0.4），不要让 1.B 和 1.D 各改一半。
- Linux：`enigo` X11 可用；Wayland/失败时降级「仅复制+关窗」。

### 6.4 时序
```
macOS：
  用户在 App A 输入框 → 按快捷键
    → QuickBar 以非激活面板弹出，App A 仍前台
    → 打字过滤 / ↑↓ 选择 / Tab 切常用
    → Enter
        → 写系统剪贴板（打 CopyMarker）
        → 隐藏 QuickBar
        → enigo 发 Cmd+V
        → App A 收到粘贴

Windows：
  用户在 App A 输入框 → 按快捷键
    → 记录 App A 前台窗口句柄
    → QuickBar 正常取焦点供输入
    → Enter
        → 写系统剪贴板（打 CopyMarker）
        → 隐藏 QuickBar
        → 恢复 App A 前台窗口
        → enigo 发 Ctrl+V
        → App A 收到粘贴
```

## 7. 核心四：置顶/常用 一等公民设计

### 7.1 为什么单独设计
「最近历史」和「常用置顶」是两种心智模型：

| | 最近历史 | 常用置顶 |
|---|---|---|
| 本质 | 易逝、临时 | 持久、精选 |
| 内容 | 刚复制的话/链接 | 邮箱、地址、签名、命令、模板 |
| 找它方式 | 顶部 / 搜索 | 凭**位置记忆**直接取 |
| 排序 | 按时间（一直变） | 按你定的固定顺序（不变） |
| 寿命 | 会自动清理 | 永不回收 |

### 7.2 三大能力
1. **稳定槽位 + 肌肉记忆（最大价值）**：常用项顺序固定（拖动排好、不随使用而变），「`Tab`→按 `2`」永远粘同一条，用熟了不用看屏幕。`pin_order` 已支持稳定排序，本期把「数字槽位」暴露到 UI。
2. **可选标签/标题**：给常用项起名（「工作邮箱」「上线命令」），列表可读、可搜。数据加 `label` 字段。等于把 ClipMan 顺手变成**轻量常用语/代码片段库**。
3. **独立入口**：`Tab` 直达；可选第二全局快捷键直达。

### 7.3 摆法：双面板（已确认方案 A）
- 默认「最近」；`Tab` 切「常用」。
- 好处：打开时最近项在最顶最快；常用槽位稳定、能形成肌肉记忆；互不挤占。
- 托盘菜单维持「置顶区在上、最近区在下」的分区样式（鼠标路径，不变）。

### 7.4 操作
- 提拔：最近项 `Cmd/Ctrl+P` → 成为常用。
- 命名：卡片「编辑标签」按钮 / 选中后某键，弹内联输入。
- 排序：常用面板 `Cmd/Ctrl+Shift+↑/↓` 或拖拽。
- **图片存储策略改为「始终存原图（2048 上限）+ 派生缩略图」**：捕获时同时存原图到 `content`、存 256px 缩略图到新列 `thumbnail`；列表/托盘渲染用缩略图，复制/粘贴用原图。这样置顶图片天然保持高清，从根上消除「先存了缩略图、置顶时无法恢复原图」的问题。原 `store_original_image` 设置废弃。代价：磁盘占用上升（受 `max_history_items` 上限约束，可控）。
- 置顶天然不被「清空历史」和自动回收影响（现状保留）。

## 8. 数据模型 🔧

去加密后内容明文存储，schema 升级如下：

```sql
clips (
  id           TEXT PRIMARY KEY,  -- UUID
  content      BLOB NOT NULL,     -- 明文原始内容（文本=UTF-8 字节；图片=原图 PNG，2048 上限）
  thumbnail    BLOB,              -- 🆕 图片 256px 缩略图（仅 image 类型；列表/托盘渲染用）
  content_hash TEXT,              -- SHA256，去重 + 自捕获标记用
  content_type TEXT NOT NULL,     -- text/image/file/html/rtf
  timestamp    INTEGER NOT NULL,
  is_pinned    INTEGER DEFAULT 0,
  pin_order    INTEGER,           -- 常用稳定排序
  label        TEXT,              -- 🆕 可选标题（主要给常用）
  group_name   TEXT               -- 🆕 预留分组，本期不用（group 是 SQL 关键字，用 group_name）
)

-- 全文索引：独立表，Rust 侧维护，rowid 对齐 clips.rowid（详见 §9.4，写死不留猜测）
clips_fts USING fts5(clip_id UNINDEXED, search_text, label, tokenize='trigram')

PRAGMA user_version = 1     -- 迁移标记
PRAGMA journal_mode = WAL   -- 读写更顺，零成本
```

索引保留 `idx_timestamp` / `idx_pinned` / `idx_content_hash`。

**最近/常用数据契约要拆清**（当前 `get_recent()` 把 pinned+recent 按 timestamp 混排，不符合双面板）：新增 `get_recent_clips(limit)`（**只返回非置顶**）与 `get_pinned_clips()`（按 `pin_order`）；搜索结果带 `is_pinned` 字段供前端分组。

## 9. 去加密 + 迁移 + FTS5 🔧

### 9.1 为什么必须去加密
1. **防不住任何现实威胁**：密钥 `.clipman.key` 明文存在数据库**同目录**（`main.rs:58-99`），能读 DB 就能读密钥——加密剧场，收益≈0。
2. **藏数据丢失风险**：解密失败**静默丢弃**该条（`storage.rs:248-277`），密钥一丢历史无声蒸发。
3. **拖累搜索**：密文无法走索引，只能全解密到内存子串匹配（`storage.rs:280-333`）。
4. **污染代码**：`Option<Arc<Crypto>>` 穿透 storage/main/migration。

### 9.2 删除清单
删 `crypto.rs`；删 `main.rs` 的密钥生成/`Crypto`/`AppState.crypto`；`ClipStorage` 去掉 crypto 字段，直接存取明文；移除 `ring` 依赖（若无他用）。

### 9.3 迁移（因「可接受重置」从简）
用 `PRAGMA user_version` 当版本标记：
```
启动时若 user_version < 1：
  若存在 .clipman.key：
    逐行尝试解密 → 明文重写 content（解密失败的行直接删，可接受）
    删除 .clipman.key
  rebuild FTS 索引
  设 user_version = 1
兜底：整体迁移失败时允许清空 clips 重来，不崩溃。
```

### 9.4 FTS5（写死设计，不留猜测）
- 表：`clips_fts USING fts5(clip_id UNINDEXED, search_text, label, tokenize='trigram')`，**`rowid` 对齐 `clips.rowid`**，靠 `clip_id` 关回主表。
- **维护放在 Rust 侧**（不用 SQL 触发器）：`insert`/`delete`/`set_clip_label` 等改动 clips 的方法里同步写/删对应 FTS 行——因为 Rust 手里有解码后的纯文本和 label，可避免在 SQL 触发器里对 BLOB 做 `CAST` 解码的坑。`search_text` 仅对文本/文件名类型填充。
- **查询长度处理**：`query` 的 Unicode 字符数 **≥3** 走 `MATCH`（对输入转义）；**<3 字符（如中文一两个字）回退 `LIKE '%q%'` 扫描**——trigram 不索引短于 3 的串，不回退会搜不到。
- 命中后用 rowid/clip_id join 回 `clips` 取完整条目。
- 迁移时 `rebuild` 回填存量。

## 10. 隐私护栏（替代加密的正解）

- **[本期] 跳过密码类剪贴板**（设置 `ignore_concealed`，默认开）：
  - macOS：检测 `org.nspasteboard.ConcealedType` / `TransientType` / `AutoGeneratedType`，命中不保存。
  - Windows：检测 `ExcludeClipboardContentFromMonitorProcessing` / `CanIncludeInClipboardHistory`，命中不保存。
- **[后续] 应用排除名单**（按前台应用 bundle id/进程名）。
- **[后续] 定时/触发清理**（退出/锁屏清空非置顶）。
- **[§10.4 将来若真要落盘加密]** 正确姿势：密钥派生自系统钥匙串（macOS Keychain / Windows DPAPI）或用户口令 + SQLCipher 整库加密。本期明确不做。

## 11. 架构取舍（响应「可不参考旧实现」）

诚实判断：**技术栈是对的，不推倒重写**（Tauri+Rust+SQLite+Svelte5 对轻量剪贴板就是最优解）。但以下按最佳实践重做：

| | 内容 | 处理 |
|---|---|---|

## 13. 分期实施（用户视角里程碑）

> 实现层的依赖顺序与并行方案见 [`DEVELOPMENT_PLAN.md`](./DEVELOPMENT_PLAN.md)。下面是「用户能感知的发版节奏」。

- **M1 取用体验质变**：QuickBar 键盘导航 + 整卡可点 + 自动粘回 + 入口分流。
  验收：编辑器里按快捷键→打两个字→回车→已粘进编辑器，全程不碰鼠标、不切窗口。
- **M2 常用 + 弹窗形态**：双面板/数字槽位/标签 + 无边框置顶居中弹窗 + Settings 独立窗口。
  验收：`Tab`→按 `2` 稳定粘出同一条常用；弹窗像 Spotlight 一样轻。
- **M3 减负 + 隐私正解**：去加密 + 迁移 + FTS5 + 跳过密码类剪贴板。
  验收：旧库平滑迁移（或可接受重置）；搜索走索引；复制密码不进历史；`crypto.rs` 已删。

## 14. 风险与注意点

- **自动粘贴健壮性**：少数应用拦截合成事件，需在 VS Code/浏览器/Word/终端/微信飞书输入框实测。
- **macOS 权限引导**：辅助功能没给时自动粘贴静默失败，要检测并提示。
- **隐藏与发键时序**：非激活面板方案从根上规避（焦点没离开过原应用）；兜底方案要保证「隐藏→焦点回原应用→发键」顺序。
- **图片自捕获**：记得给图片补 `last_copied_by_us` 标记（§6.3）。
- **FTS5 存量回填**：迁移时 rebuild 一次索引。
- **`group` 关键字**：列名用 `group_name`。
